Change Log

All notable changes to this project will be documented in this file. This project adheres to Semantic Versioning. This change log adheres to standards from Keep a CHANGELOG.

Unreleased

2.2.0 - 2016-11-07

Fixed

  • Corrected a few gaffs in the auto-ignore logic to fix major performance issues with projects that did not explicitly ignore node_modules. (#654)
  • import/ignore setting was only being respected if the ignored module didn't start with an import or export JS statement
  • [prefer-default-export]: fixed crash on export extensions (#653)

2.1.0 - 2016-11-02

Added

  • Add ./docs/rules/no-named-default.md "`no-named-default`" rule: style-guide rule to report use of unnecessarily named default imports (#596, thanks )
  • [no-extraneous-dependencies]: check globs against CWD + absolute path (#602 + #630, thanks )

Fixed

  • ./docs/rules/prefer-default-export.md "`prefer-default-export`" handles flow export type (#484 + #639, thanks )
  • ./docs/rules/prefer-default-export.md "`prefer-default-export`" handles re-exported default exports (#609)
  • Fix crash when using ./docs/rules/newline-after-import.md "`newline-after-import`" with decorators (#592)
  • Properly report ./docs/rules/newline-after-import.md "`newline-after-import`" when next line is a decorator
  • Fixed documentation for the default values for the ./docs/rules/order.md "`order`" rule (#601)

2.0.1 - 2016-10-06

Fixed

  • Fixed code that relied on removed dependencies. (#604)

2.0.0! - 2016-09-30

Added

  • ./docs/rules/unambiguous.md "`unambiguous`" rule: report modules that are not unambiguously ES modules.
  • recommended shared config. Roughly errors and warnings mixed together, with some parserOptions in the mix. (#402)
  • react shared config: added jsx: true to parserOptions.ecmaFeatures.
  • Added ./docs/rules/no-webpack-loader-syntax.md "`no-webpack-loader-syntax`" rule: forbid custom Webpack loader syntax in imports. (#586, thanks !)
  • Add option newlines-between: "ignore" to order
  • Added ./docs/rules/no-unassigned-import.md "`no-unassigned-import`" rule (#529)

Breaking

  • import/extensions setting defaults to ‘[’.js']. ([#306])
  • [import/ignoresetting] defaults to nothing, and ambiguous modules are ignored natively. This means importing from CommonJS modules will no longer be reported by [default], [named], or [namespace], regardless ofimport/ignore. ([#270])
  • [newline-after-import]: Removed need for an empty line after an inlinerequirecall ([#570])
  • [order]: Default value fornewlines-betweenoption is nowignore` (#519)

Changed

  • imports-first is renamed to ./docs/rules/first.md "`first`". imports-first alias will continue to exist, but may be removed in a future major release.
  • Case-sensitivity: now specifically (and optionally) reported by ./docs/rules/no-unresolved.md "`no-unresolved`". Other rules will ignore case-mismatches on paths on case-insensitive filesystems. (#311)

Fixed

  • [no-internal-modules]: support @-scoped packages (#577+#578, thanks )

1.16.0 - 2016-09-22

Added

  • Added ./docs/rules/no-dynamic-require.md "`no-dynamic-require`" rule: forbid require() calls with expressions. (#567, #568)
  • Added ./docs/rules/no-internal-modules.md "`no-internal-modules`" rule: restrict deep package imports to specific folders. (#485, thanks !)
  • [extensions]: allow override of a chosen default with options object (#555, thanks !)

Fixed

  • ./docs/rules/no-named-as-default.md "`no-named-as-default`" no longer false-positives on ‘export default from ’...'([#566], thanks [@preco21])
  • [default`]: allow re-export of values from ignored files as default (#545, thanks )

  • ./docs/rules/max-dependencies.md "`max-dependencies`" for specifying the maximum number of dependencies (both import and require) a module can have. (see #489, thanks )

1.3.0 - 2016-03-20

Major perf improvements. Between parsing only once and ignoring gigantic, non-module node_modules, there is very little added time.

My test project takes 17s to lint completely, down from 55s, when using the memoizing parser, and takes only 27s with naked babel-eslint (thus, reparsing local modules).

Added

  • This change log (#216)
  • Experimental memoizing ./memo-parser/README.md "parser"

Fixed

  • Huge reduction in execution time by only ignoring import/ignore setting if something that looks like an export is detected in the module content.
