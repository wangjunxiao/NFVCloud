Offers a async require.\+resolve function. It\textquotesingle{}s highly configurable.

\subsection*{Features}


\begin{DoxyItemize}
\item plugin system
\item provide a custom filesystem
\item sync and async node.\+js filesystems included
\end{DoxyItemize}

\subsection*{Contributing}

Take a look at {\ttfamily lib/\+Resolve\+Factory.\+js} to understand how everything fits together. It provides a {\ttfamily create\+Resolver(options)} function and these are the possible options\+:

\tabulinesep=1mm
\begin{longtabu} spread 0pt [c]{*{3}{|X[-1]}|}
\hline
\rowcolor{\tableheadbgcolor}\textbf{ Field  }&\textbf{ Default  }&\textbf{ Description   }\\\cline{1-3}
\endfirsthead
\hline
\endfoot
\hline
\rowcolor{\tableheadbgcolor}\textbf{ Field  }&\textbf{ Default  }&\textbf{ Description   }\\\cline{1-3}
\endhead
modules  &\mbox{[}\char`\"{}node\+\_\+modules\char`\"{}\mbox{]}  &A list of directories to resolve modules from, can be absolute path or folder name   \\\cline{1-3}
description\+Files  &\mbox{[}\char`\"{}package.\+json\char`\"{}\mbox{]}  &A list of description files to read from   \\\cline{1-3}
plugins  &\mbox{[}\mbox{]}  &A list of additional resolve plugins which should be applied   \\\cline{1-3}
main\+Fields  &\mbox{[}\char`\"{}main\char`\"{}\mbox{]}  &A list of main fields in description files   \\\cline{1-3}
alias\+Fields  &\mbox{[}\mbox{]}  &A list of alias fields in description files   \\\cline{1-3}
main\+Files  &\mbox{[}\char`\"{}index\char`\"{}\mbox{]}  &A list of main files in directories   \\\cline{1-3}
extensions  &\mbox{[}\char`\"{}.\+js\char`\"{}, \char`\"{}.\+json\char`\"{}, \char`\"{}.\+node\char`\"{}\mbox{]}  &A list of extensions which should be tried for files   \\\cline{1-3}
enforce\+Extension  &false  &Enforce that a extension from extensions must be used   \\\cline{1-3}
module\+Extensions  &\mbox{[}\mbox{]}  &A list of module extensions which should be tried for modules   \\\cline{1-3}
enforce\+Module\+Extension  &false  &Enforce that a extension from module\+Extensions must be used   \\\cline{1-3}
alias  &\mbox{[}\mbox{]}  &A list of module alias configurations or an object which maps key to value   \\\cline{1-3}
resolve\+To\+Context  &false  &Resolve to a context instead of a file   \\\cline{1-3}
unsafe\+Cache  &false  &Use this cache object to unsafely cache the successful requests   \\\cline{1-3}
cache\+With\+Context  &true  &If unsafe cache is enabled, includes {\ttfamily request.\+context} in the cache key   \\\cline{1-3}
cache\+Predicate  &function() \{ return true \};  &A function which decides whether a request should be cached or not. An object is passed to the function with {\ttfamily path} and {\ttfamily request} properties.   \\\cline{1-3}
file\+System  &&The file system which should be used   \\\cline{1-3}
resolver  &undefined  &A prepared Resolver to which the plugins are attached   \\\cline{1-3}
\end{longtabu}


The options are passed from the {\ttfamily resolve} key of your webpack configuration e.\+g.\+:


\begin{DoxyCode}
resolve: \{
  extensions: ['', '.js', '.jsx'],
  modules: ['src', 'node\_modules'],
  plugins: [new DirectoryNamedWebpackPlugin()]
  ...
\},
\end{DoxyCode}
